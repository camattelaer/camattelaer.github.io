## Step 1: install `ruby`

Since I am currently working from my media server which runs Windows 11, the install steps are (currently?) limited for Windows only. In the future, I might add for Mac and Linux, but installing `ruby` is normally rather trivial so you could find other sources to help you along.

Firstly, download the `RubyInstaller` executable from this [link](https://rubyinstaller.org/downloads/). Execute the installer like you would for other Windows programs and make sure to select the option to add the install folder to `PATH`. If the installer has finished, you should open a terminal (quick access by `Windows key + r` and enter `cmd` in the dialog box). In the terminal, enter `ruby -v` to check if the binary can be found.

## Step 2: install `ruby` dependencies

With `ruby` installed, the only dependency we need to install is `bundle`. So run `gem install bundle`. `bundle` will require a `Gemfile` to be present in your repo root. If this is not present create a `Gemfile` with the following content:
```
source "https://rubygems.org"

gem "github-pages", group: :jekyll_plugins
```

If a `Gemfile` is present, you can build the dependencies using `bundle install` from the repo root.

## Step 3: run the website
After installing the necessary dependencies, the website can be launched via `bundle exec jekyll serve`. This will by default use `localhost:4000` to host the website. In my particular case, port 4000 was not available so I simply routed to port 4001 by executing `bundle exec jekyll serve --port 4001`. Normally, this will launch your website if the port is available and all you have to do to visit the website is browse to `http://localhost:<PORT>` to visualize the website, see Figure 2.
