## on the old controller

1. stop SLURM and corresponding services:
```
sudo scontrol update NodeName=<HOSTNAME> State=DRAIN Reason="Migrating controller"

# Then stop the controller
sudo systemctl stop slurmctld
sudo systemctl disable slurmctld
```
2. if you want to preserve the job history and node states, you need to copy over the folder indicated by `StateSaveLocation` (typically in `/var/spool/slurmctld/` or something similar) in the `/etc/slurm/slurm.conf/` file. You can do this using `rsync` for example:
```
sudo rsync -av /var/spool/slurmctld/ <NEW_CONTROLLER_HOSTNAME>:/var/spool/slurmctld/
```

## On the new control node

1. install `slurmctld`: `$sudo apt install slurmctld`
  (include `slurmd` if you want to include it as a compute node in the cluster)
2. if you did not copy the `StateSaveLocation` from the oldcontroller (Step 2 in ), then we need to create this location:
```
sudo mkdir -p /var/spool/slurmctld
sudo chown slurm:slurm /var/spool/slurmctld
sudo chmod 755 /var/spool/slurmctld
```
3. update `/etc/slurm/slurm.conf` to reflect the new controller. Make sure the following variables are changed (if defined):
  - `SlurmctldHost`
  - `SlurmctldAddress`
  - `StateSaveLocation` (if modified compared from original controller)
4. Distribute the `slurm.conf` file to compute nodes:
```
#!/bin/bash

while IFS= read -r node; do
  echo " trying node='${node}'"
  if ssh -o ConnectTimeout=5 -o BatchMode=yes ${node} true </dev/null 2>/dev/nu>
    scp /etc/slurm/slurm.conf ${node}:/tmp/slurm.conf </dev/null
    ssh ${node} "sudo mv /tmp/slurm.conf /etc/slurm/slurm.conf" </dev/null
    echo "${node}: config updated"
  else
    echo "${node}: unreachable, skipping"
  fi
done < nodelist.txt
```
  - please not this requires passwordless ssh access using private/public keys, eg:
```
ssh-keygen -t ed25519 -C "slurm management key"

# copy to all nodes
ssh-copy-id camattelaer@mapo-xeon
ssh-copy-id camattelaer@mapo-gpu01
```
  - this also requires `sudo` to copy to the write-protected `slurm.conf` file. This is not possible during regular `ssh`/`scp` so you need to setup a `sudoers` rule on each node, eg:
```
ssh <NODE>

echo '<USER> ALL=(ALL) NOPASSWD: /bin/mv /tmp/slurm.conf /etc/slurm/slurm.conf' | sudo tee /etc/sudoers.d/slurm-conf
```
