For v2 of the cluster, I bought an RTX Pro 4000 card. The RTX Pro 4000 has a brandspanking new Blackwell architecture. Several issues came up after install the card:
1) Black screen on boot. It displayed the startup sequence from motherboard and rendered the BIOS so it had to be a Linux specific issue and not hardware related. I found this post on [Reddit](https://www.reddit.com/r/comfyui/comments/1li8cly/troubleshooting_nvidia_driver_installation_for_a/) which was the exact same issue! The solution? Open source drivers was al it took to correct this error.
2) When trying the RTX Pro 4000 for AMBER, I encountered an obsure error I had never seen before: `cudaMemcpyToSymbol: SetSim copy to cSim failed no kernel image is available for execution on the device`. AMBER worked perfectly on the two RTX 4000 ADA cards (older architecture). I did not have to find long for a solution as it appeared quite easily when searching for this particular error in Google. As explained in this [link](https://forums.developer.nvidia.com/t/amber24-on-rtx5090/328465/3), you need to modify some of the config files. Just for the sake of completeness, I only had 1 of those files in the `src` present (`amber24_src/cmake\CudaConfig.cmake`) and the entry on line 17 of that particular file was `set(SM120FLAGS -gencode arch=compute_120,code=sm_120)` simply changing this line to `set(SM120FLAGS -gencode arch=compute_120,code="compute_120,sm_120")` was not enough. Lines 19 and 21 had similar content but `100` and `90` instead of `120`, respectively. I modified those two lines in the same way and now `pmemd.cuda` DID startup on the RTX Pro 4000. Great success! -->
